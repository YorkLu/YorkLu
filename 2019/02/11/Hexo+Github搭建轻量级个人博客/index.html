Hexo+Github搭建轻量级个人博客

发表于 2019-02-11

前言

大概CS相关专业的小伙伴/小妞伴都曾有过建个人网站的想法，不少人早已付诸实践，而我由于懒癌加上总觉得搭服务器的学习曲线会相当陡峭（立个flag：希望今年能克服惰性，搭一个有后端服务的应用），一直只是想想而已。

节前刚好看到一个利用github pages支持静态页解析、实现本地文件发布到github pages页面的博客框架——hexo，算是把博客搭了起来，第一篇文章就用来回顾下踩坑的过程——事实上，用hexo搭个人博客的过程并不需要“写代码”，所以非CS专业的小伙伴也可以大胆撸起来，体验从0到1快速搭建个人博客的过程~

用到的工具

Github & Github Pages

Github是全世界最大的程序猿交友网站一个软件项目托管平台，而github pages是github提供给开发者用于生成项目网站的工具——简单来说，github允许我们上传代码，并基于这些代码生成可以访问的静态页面。

不对啊，不是刚说了不需要“写代码”吗？网站的代码可以通过别的工具生成，这边是下面要介绍的hexo了↓↓↓

Hexo

Hexo是一个生成静态（html+css+js）网站的工具，能够把markdown格式的文章批量转化为网站，Hexo的安装方式后面会详细介绍，我们先了解一下markdown格式↓

Markdown

Markdown 是一种标记语言，允许我们用纯文本格式编写文档，最终显示为带有丰富格式的HTML页面，具体效果可以看图↓

Markdown语言常用语法不多，很多编辑器也支持类Word的操作模式，及原始文本与效果对照显示，这里推荐一个在线编辑器——Cmd Markdown。

Git

了解了前面三个工具，用hexo+github搭建及编辑个人博客的流程已经基本出来了——

graphTD;
    A[开始] –>|打开编辑器| B(用Markdown写文章);
    B–>|输入一些命令|C(用Hexo生成静态页);
    C–>|如何发布? 用Git|D(把静态页发布到Github上);
    D–>E(搞定);

Git便是把我们在本地生成的静态网站发布到github的工具，它需要跟我们的Github账号进行绑定。下面开始介绍详细的搭建步骤↓
